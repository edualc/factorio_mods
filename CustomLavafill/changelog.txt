Version: 2.1.4
Date: 2026-07-20
  Bugfixes:
    - Fixed offshore pumps sometimes not starting when placed together with lavafill in the same
      blueprint, requiring the player to remove and re-place the pump manually. An offshore pump
      resolves its connection to the fluid tile beneath it once, at creation time; construction
      robots can finish building the pump before the lava tile underneath it is actually built,
      leaving it permanently unconnected. control.lua now listens for the lava tile itself being
      built (on_robot_built_tile / on_player_built_tile / on_space_platform_built_tile) and, if an
      offshore pump is already sitting on that tile, destroys and recreates it (preserving position,
      direction, force, and quality) so it re-resolves its connection against the now-correct tile.

Version: 2.1.3
Date: 2026-07-19
  Bugfixes:
    - Fixed direct player placement of lavafill still failing on resource-covered tiles despite the
      2.1.2 fix: on_pre_build fires before the engine resolves the current build attempt, so the
      current click's placement validity already appears to be decided by the time the handler runs
      - destroying the resource there only unblocked the next click on the same tile, not the one in
      progress. on_pre_build now places the tile itself via LuaSurface.set_tiles (which also drops
      any colliding entity) and consumes the item by hand whenever a resource is found, instead of
      destroying the resource and relying on the native place_as_tile follow-up to pick it up. The
      blueprint/construction-robot path from 2.1.2 is unaffected: robots build on a later tick, well
      after the ghost-creation-time resource removal, so it already resolves without this timing issue.

Version: 2.1.2
Date: 2026-07-19
  Bugfixes:
    - Fixed lavafill failing to place on tiles with a resource entity on them (ore, oil, etc.):
      resource entities get the engine-default collision_mask {layers={resource=true}} unless a
      prototype overrides it, and place_as_tile.condition only allows overriding collision layers
      it explicitly lists. The condition previously listed only lava_tile, so it could place lava
      onto existing lava tiles but never onto ore patches. Added resource to the condition layers.
    - Belt-and-suspenders fix for the same issue: added control.lua to destroy any resource entity
      under the target tile before placement, guaranteeing placement succeeds regardless of the
      exact place_as_tile.condition behavior. Covers both placement paths: direct player placement
      (destroys the resource on on_pre_build, before the engine's own check runs) and blueprint /
      construction-robot placement, which always goes through a tile-ghost first (destroys the
      resource as soon as the lavafill tile-ghost is created, since robots have no pre-build hook
      of their own).
